## Tone of Voice

### How DataForge Should Sound
| Dimension | Description |
|-----------|-------------|
| **Professional** | Credible, authoritative, trustworthy (not cute or casual) |
| **Empowering** | Action-oriented, confident, "you can do this at scale" |
| **Technical** | Precise terminology, respect for expertise |
| **Bold** | "Steroids and superpowers" — confident in capabilities |
| **Accessible** | Explain technical features in benefit language |
| **Helpful** | Guides users to success, not lecturing |

### Example Messaging
❌ **Too casual:** "Hey, let's delete some duplicate files, lol"  
✅ **Right tone:** "Intelligently identify and remove duplicate files at scale with one command"

❌ **Too vague:** "This tool is pretty good at file stuff"  
✅ **Right tone:** "DataForge uses cryptographic hashing and parallel processing to find exact duplicates across millions of files in seconds"

❌ **Too technical:** "SHA-256 hexadecimal digest collision detection"  
✅ **Right tone:** "Cryptographic verification ensures you're finding exact duplicates, not similar-looking files"

## Avoid These Mistakes

❌ **Overusing "superpowers"** — Use it 1-2x per page max, not in every sentence  
❌ **Being too casual** — "Steroids and superpowers" is bold, but tone stays professional  
❌ **Hiding the tagline** — It should appear in GitHub, setup.py, About dialog, and landing page  
❌ **Contradicting safety** — Talk about batch operations power, but always emphasize dry-run previews  
❌ **Ignoring the audience** — Adjust messaging for forensics vs. cleanup vs. developers  
❌ **Claiming unverified features** — Only market what's in code and tested (224 tests ✓)

## Questions?

**Q: Is "steroids and superpowers" too playful for an enterprise tool?**  
A: No — it's bold and confident, which signals that DataForge is *actually* powerful, not just claiming to be. Enterprise users respond to confidence and capability. It's cheeky but professional.

**Q: Should we use "steroids" in legal/compliance messaging?**  
A: For marketing and product, yes. For security audits or compliance docs, use "enterprise-grade" or "production-tested" instead. Context matters.

**Q: Can we change the tagline later?**  
A: Yes, but consistency now builds brand recognition. Stick with it for at least 6-12 months before reconsidering.
